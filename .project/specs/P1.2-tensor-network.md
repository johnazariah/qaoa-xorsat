## Goal

Define the tensors that represent each component of the QAOA "sandwich"
⟨γ,β|C_α|γ,β⟩ on the light-cone tree, and the contraction structure that
combines them. This spec translates the quantum circuit into a concrete tensor
network.

## Background

The QAOA expectation value for the root constraint α is:

$$\langle \gamma, \beta | C_\alpha | \gamma, \beta \rangle$$

where |γ,β⟩ = U(B,βₚ)U(C,γₚ)···U(B,β₁)U(C,γ₁)|+⟩^⊗n.

This is a "bra-ket sandwich" which, when expanded, gives a tensor network with:
- One copy of the circuit for the ket
- The observable C_α in the middle
- One copy of the conjugated circuit for the bra

**Key simplification**: Since the problem (C) gates are diagonal in the Z basis,
and the initial state |+⟩ is separable, and O = Z_{i₁}···Z_{i_k} is also
diagonal, we can use a **hyperindex** representation where each qubit contributes
a single 2p-component binary index rather than separate bra/ket indices.

This is the insight from Farhi et al. 2025 (Eq. 14 and surrounding discussion).

## The Hyperindex Representation

For each qubit q, define a **hyperindex** $\sigma_q = (\sigma_q^1, \ldots, \sigma_q^{2p})$
where $\sigma_q^\ell \in \{0, 1\}$. The full hyperindex has 2p bits, so $2^{2p} = 4^p$
possible values.

The 2p components correspond to the 2p "time slices" of the bra-ket sandwich:
- Components 1, ..., p correspond to the ket circuit layers (forward in time)
- Components p+1, ..., 2p correspond to the bra circuit layers (backward in time)

More precisely: at each QAOA round ℓ, the problem gate acts diagonally (just
phases), and the mixer gate $e^{-iβ_ℓ X}$ is the only gate that mixes computational
basis states. So the hyperindex $\sigma_q$ tracks the "path" through bases
at each mixer application.

### Tensors in this representation

**1. Leaf tensor (initial state |+⟩, combined bra-ket)**

Each leaf qubit starts in |+⟩. The combined bra-ket contribution is:

$$L(\sigma) = \prod_{\ell=1}^{p} M_{\sigma^{2\ell-1}, \sigma^{2\ell}}(\beta_\ell)$$

where $M$ is the 2×2 mixer "transition" matrix (to be derived precisely).

This is a vector of length $4^p$ — one entry per hyperindex value.

**2. Problem gate tensor (k-body diagonal phase)**

For a constraint node connecting variables $q_1, \ldots, q_k$, at QAOA round ℓ
with angle γ_ℓ, the contribution to the sandwich is:

$$P(\sigma_{q_1}, \ldots, \sigma_{q_k}; \gamma_\ell) = \exp\!\Big(-i\gamma_\ell \cdot \frac{(-1)^{\sigma_{q_1}^\ell \oplus \cdots \oplus \sigma_{q_k}^\ell}}{2}\Big) \cdot \text{(conjugate for bra)}$$

Since the gate is diagonal, the bra and ket contributions multiply, giving a
real-valued result (the phases combine).

**3. Mixer tensor (single-qubit X rotation)**

The mixer $e^{-i\beta X}$ is the only term that couples different hyperindex
components. Its contribution for qubit q at round ℓ is the transition amplitude
between consecutive hyperindex bits.

**4. Observable tensor (root constraint)**

For the root constraint, the observable $C_\alpha = (1 + Z_{i_1}\cdots Z_{i_k})/2$
contributes a tensor at the root that depends on the parity of the root qubits'
hyperindices.

### Angle parameters

```julia
"""
QAOA angle parameters for depth p.

- `γ`: problem angles, length p
- `β`: mixer angles, length p
"""
struct QAOAAngles
    γ::Vector{Float64}
    β::Vector{Float64}

    function QAOAAngles(γ::Vector{Float64}, β::Vector{Float64})
        length(γ) == length(β) ||
            throw(ArgumentError("γ and β must have same length"))
        length(γ) ≥ 1 ||
            throw(ArgumentError("need at least p=1"))
        new(γ, β)
    end
end

depth(a::QAOAAngles) = length(a.γ)
```

### Hyperindex utilities

```julia
"""Extract bit ℓ (1-indexed) from a 2p-bit hyperindex σ."""
hyperindex_bit(σ::Int, ℓ::Int) = (σ >> (ℓ - 1)) & 1

"""Compute the parity of bits at positions given by `rounds` in hyperindex σ."""
hyperindex_parity(σ::Int, rounds) = reduce(⊻, hyperindex_bit(σ, ℓ) for ℓ in rounds)
```

## Mathematical Details to Derive

The precise form of each tensor requires working through the algebra carefully.
This is the core mathematical work of Phase 1. Specifically:

1. **Derive the leaf tensor** — expand |+⟩ through p rounds of mixer gates in the
   sandwich picture. This should be expressible as a product over rounds.

2. **Derive the problem gate contribution** — express the 3-body gate
   $\exp(-i\gamma Z_1 Z_2 Z_3 / 2)$ in the hyperindex basis. For k=2 (MaxCut),
   this is well-known from Farhi 2025. For k=3, the parity involves 3 qubits
   instead of 2 — the structure is analogous but the combinatorics differ.

3. **Derive the mixer contribution** — the mixer $e^{-i\beta X}$ couples adjacent
   hyperindex bits. This is the same for any k — it's a single-qubit gate.

4. **Verify dimensional consistency** — the contracted result must be a scalar
   (the expectation value), and each intermediate tensor must have the correct
   index structure.

**Reference**: Farhi et al. 2025, Section III and Appendix A (for k=2). We
generalise to k=3.

combination eliminates complex phases). This must be explicitly asserted:

```julia

```

### 7. Zero-angle identity test

At γ=β=0, the QAOA circuit is the identity (no phases, no mixing beyond the
initial |+⟩ state). The resulting expectation value should be 0.5 (random
guess). This constrains what the tensors look like at zero angles:

```julia
@testset "zero_angle_tensors" begin
    for p in 1:3
        angles = QAOAAngles(zeros(p), zeros(p))
        L = leaf_tensor(angles)
        # At zero angles, the leaf tensor should have a specific known form
        # (all entries equal, since the mixer at β=0 is the identity)
        @test all(x -> x ≈ L[1], L)
    end
end
```

### 8. Periodicity tests

```julia
@testset "tensor_periodicity" begin
```

## Acceptance Criteria

1. All tensor functions return arrays of the correct size (4^p)
2. Hyperindex utilities are correct for known bit patterns
3. At zero angles (γ=β=0), tensors reduce to known identity-circuit values
4. All tensors are real-valued (not complex) — asserted explicitly
5. Hand-derived golden values for p=1, k=2 match implementation to 1e-12
6. Hand-derived golden values for p=1, k=3 match implementation to 1e-12
7. Tensors are 2π-periodic in both γ and β
8. Mathematical derivation is documented in `.project/learning/05-tensor-derivation.md`
