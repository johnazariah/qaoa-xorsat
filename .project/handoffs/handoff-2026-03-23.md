## Key Learnings / Gotchas

1. **g_abstol fix**: Optim's default `g_abstol=1e-8` is unreachable with 
   finite-difference gradients on the k=3 D=4 landscape. We set `g_abstol=1e-6`
   in `src/optimization.jl:234`. This completely fixed p=5 convergence
   (400 iters → 17 iters). The value didn't change — the optimizer was already
   at the right answer, it just couldn't satisfy the gradient tolerance.

2. **Adaptive budget**: `depth_optimization_budget()` in `src/optimization.jl`
   scales restarts down and maxiters up for higher p. For p≥5 it uses 
   `min(restarts, 2)` and `4 * maxiters`.

3. **Retry mechanism**: `optimize_depth_sequence()` retries with doubled maxiters
   if a warm-started depth doesn't converge. With the g_abstol fix this should
   rarely trigger.

4. **Terminal cwd**: The `bash -lc` subshell in `run-with-machine-state.sh`
   starts in a different directory. Always include `cd /workspace` inside the
   subshell command. The runbook already has this.

5. **Old worktree**: `/workspace/.worktree/phase4-optimization/` is a dead
   artifact from the merged feature branch. Ignore it. There's also a
   `.worktree/` entry in `.gitignore`.

6. **Results directory**: `results/` at the workspace root is gitignored and
   holds machine-state snapshots. Preserved optimization data goes to
   `.project/results/optimization/` (committed). The `index.csv` there has
   all historical runs.

7. **Do NOT run p≥9 on this box** — the runbook prohibits it. That's for
