   - Validates: zero-angle baseline, MaxCut p=1 formula, MaxCut p=1 optimum (0.6924), MaxCut p=2 cross-reference

   - Julia port of Basso/Villalonga compact MaxCut transfer recursion
   - Internal only — not equivalent to finite-D exact evaluator
   - Regression-locked against upstream reference values

3. **Basso finite-D iteration** (`src/basso_finite_d.jl`)
   - Implements Basso 2021 Eq. 8.7 helpers: Γ vector, f(a) function, branch tensor iteration
   - Root decomposition in progress

4. **WHT factorisation** (`src/wht.jl`)
   - **Key discovery**: constraint fold is a convolution on Z₂^{2p+1}
   - Walsh-Hadamard transform reduces cost from O(4^{3p}) to O(p²·4^p)
   - Numerically verified at p=1,2,3 for D=3,4 to machine precision (<10⁻¹⁴)
   - Documented in `learning/15-wht-factorisation-discovery.md`

5. **Transfer oracles** (`src/transfer_oracles.jl`)
   - Cross-validation infrastructure: Tier 2 must match Tier 1

#### Critical finding

The original P1.3 spec's O(4^p) claim for k>2 was wrong (constraint nodes
combine multilinearly, not entrywise). The Basso naive iteration costs O(4^{3p}).
The WHT factorisation recovers O(p²·4^p) — exact at any finite D, feasible to
p=15-17. This is a novel result not in the published literature.

tensor iteration works but wiring it to the root observable to produce the
final scalar is in progress. The developer agent is actively working on this.
